## 1. 실험 설계

| 항목 | 내용 |

| MAEUM 측 | 풀파이프라인 (MoE 프롬프트 라우팅·세션기억·KV캐시·stop방어) + MLX 엔진 |

## 2. 집계 (중앙값, n=13 — 캐시히트 1건 제외)

| 패널 | TTFT | tok/s | 출력 토큰 | 출력 범위 |
|---|---|---|---|---|
| MAEUM·Qwen | 13.9s | 12.2 | 884 | 334–1483 |
| MAEUM·Gemma | 11.2s | 16.7 | 1509 | 174–3041 |
| Ollama·Qwen | 3.6s | 16.7 | 912 | 66–1022 |
| Ollama·Gemma | 27.9s | 5.4 | 256 | 31–451 |

### 핵심 관찰

1. **🏆 Gemma 대비가 결정적** — raw Ollama Gemma는 중앙값 TTFT 27.9s · 5.4 tok/s · 256토큰으로 일관 저하, **13건 중 9건에서 문장 중간 절단 의심**. 같은 급 MAEUM Gemma는 중앙값 1,509토큰(전 패널 최장) · 16.7 tok/s로 14건 전부 완주.
2. **⚖️ Qwen TTFT는 raw가 우세 (정직 기록)** — 3.6s vs 13.9s. 풀파이프라인의 컨텍스트 조립 비용. 생성 속도(tok/s)도 raw가 다소 우세. → 개선 과제(KV 프리픽스 캐시 고도화).
3. **⚡ 응답 캐시**: 반복 쿼리 1건에서 MAEUM 양 패널 TTFT ~0초·179 t/s (캐시 반환) — 공정성 위해 집계 제외, 기능으로는 유효.

## 3. 정성 관찰 (행동 배터리에서 — 원문은 runs/ 참조)

| 쿼리 | MAEUM | raw Ollama |
|---|---|---|
| 시간·날짜 질문 | ✅ 양 모델 모두 정확 (2026-07-16 목, 03:35) | ❌ Qwen: **가짜 날짜 단정**("2025년 4월 5일 금요일") / Gemma: 모른다(정직하나 무능력) |
| 정체성 | ✅ Qwen: 설정 정체성(Assistant) 준수 / ⚠️ Gemma: 시각은 정확하나 "Google DeepMind 개발" 자칭 누수 → **미결 과제로 기록** | ❌ 각자 "Alibaba Qwen"/"Gemma 4" 자칭 |
| 가짜 전제 (없는 신작 소설) | ✅ 양 모델 "확인 불가/존재하지 않음" 거부 | ❌ Qwen: **가짜 작가명("김현식") 창작** / Gemma: 거부(정직) |
| 가짜 API (json.super_parse) | ✅ 거부 | ✅ 거부 (4패널 전원 정답 — 모델 자체 강점) |

## 4. 한계 (읽고 판단할 것)

- 각 쿼리 **1회 실행** — LLM 출력은 확률적. LRU 쿼리만 4회 반복(한/영 포함)됐고 Gemma 대비 패턴은 4회 모두 재현됨.
- raw Ollama Gemma 저하의 **근본 원인은 미규명** (서빙 기본 설정/템플릿/양자화 조합 가능성). "이 조건의 이 실행에서 이랬다"까지만 주장한다.
- 절단 감지는 휴리스틱(종결부호 미종료) — 수치는 참고치.
- 계측은 클라이언트(브라우저) 기준 — 4패널 동일 지점이므로 상호 비교엔 공정.
