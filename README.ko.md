# MAEUM Runtime — 벤치마크

[English](README.md) | **한국어**

> **"품질은 모델 크기가 아니라 런타임이 만든다."**

**MAEUM Runtime**(Apple Silicon MLX 기반 온디바이스 sLLM 런타임)의 공개 벤치마크 저장소.
작은 모델을 풀파이프라인(동적 프롬프트 라우팅·세션기억·KV 프리픽스 캐시·추론모드·그라운딩 강제)으로
감싸 대형 AI처럼 답하게 한다. 100% 로컬 — 데이터는 기기를 떠나지 않는다.

이 저장소는 주장의 근거가 되는 **원본 기록 전부**를 공개한다: 응답 전문, 패널별 계측
(TTFT / tok/s / 총시간), 그리고 우리가 진 지표까지 담은 정직한 분석.

## 🏆 대표 결과 (2026-07-16)

같은 기기(M5 MacBook) · 같은 모델 계열 · **같은 4bit 양자화** · 같은 쿼리.
MAEUM Runtime vs raw Ollama, 4패널 동시 스트리밍. 14회 실행 전 기록 공개.

**집계 (중앙값, n=13 — 캐시히트 1건 공정성 위해 제외):**

| 패널 | TTFT | tok/s | 출력 토큰 | 완주율 |
|---|---|---|---|---|
| **MAEUM · Gemma** (E2B, 4bit) | 11.2s | 16.7 | **1,509** | **14/14** |
| raw Ollama · Gemma (동급 4bit) | 27.9s | 5.4 | 256 | 절단 의심 9/13 |
| MAEUM · Qwen (4B, 4bit) | 13.9s | 12.2 | 884 | 14/14 |
| raw Ollama · Qwen (동급 4bit) | **3.6s** | 16.7 | 912 | 13/13 |

**행동 비교 (동일 쿼리 — 전문은 [`data/.../runs/`](data/20260716_maeum_vs_ollama/runs/)):**

| 테스트 | MAEUM | raw |
|---|---|---|
| "지금 몇 시야?" | ✅ 정확한 날짜·시각 | ❌ **가짜 날짜를 단정** |
| 존재하지 않는 신작 질문 | ✅ "확인되지 않음" 거부 | ❌ **가짜 작가명 창작** |
| 가짜 API (`json.super_parse`) | ✅ 거부 | ✅ 거부 |

**정직 기록**: 첫 토큰 속도(TTFT)는 raw Qwen이 이긴다(3.6s vs 13.9s) — 풀파이프라인 조립
비용이며 1순위 개선 과제. 우리가 진 지표도 공개한다. 그래야 이긴 지표가 믿긴다.

## 방법론

- **아레나**: MAEUM 내장 4패널 비교 페이지 — 쿼리 하나로 4패널 동시 발사, 계측은 4패널 동일 지점(클라이언트).
- **공정성 통제**: 계열별 동급 4bit(`qwen3_4b` MLX ↔ `qwen3:4b-q4_K_M` / `gemma-4-e2b-it` OptiQ ↔ `gemma4:e2b-it-q4_K_M`), 웹검색 OFF, 캐시히트 1건 집계 제외·표기.
- **배터리**: 행동 쿼리 10종(시간인식·추론함정·할루방어·한국어 구조화·반복·롤침범) + 코딩 과제 반복(한/영).
- **기록**: 매 실행 JSON 자동 저장 — 프롬프트·4패널 전문·추론 스트림·TTFT·토큰수·tok/s·태그·상태.

**한계** (보고서에도 명시): 쿼리당 1회 실행(확률적 — 코딩 과제만 4회 반복, 패턴 동일) ·
raw Gemma 저하의 근본 원인 미규명 · 절단 감지는 휴리스틱.
